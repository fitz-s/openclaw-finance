# RALPLAN Phase 9: Decision-Dense Campaign Surfaces

Status: approved_for_implementation
Go for implementation: true

## Task Statement

Improve campaign board, thread seed, and deep-dive cache quality so Discord surfaces provide actionable review information instead of raw internal state labels.

The current problem is structural: CampaignProjection and Undercurrent surfaces expose phrases like `unknown discovery direction conflict` and generic attention-slot language without explaining affected objects, directional implication, evidence, or what would change the decision.

## Current Failure Pattern

Bad card pattern:

```text
未知发现方向冲突（18次） | review
为什么现在：未知发现方向冲突（18次） 持续累积，需要判断是否影响 attention slot
为什么还不是动作：暗流仍在积累；未形成 isolated wake 或执行结论
与当前 book：peacetime/risk board 可见；用于避免暗流完全沉入后台
确认点：下一次 watcher update 是否同向; 相关 scenario/capital edge 是否增强
```

Why it fails:
- It states a counter, not a market object.
- It does not say which symbols/themes are implicated.
- It does not distinguish positive/negative implications.
- It does not tell the operator what to verify first.
- Thread entry is just a menu, not a prepared brief.

## Principles

1. Object first: every campaign card must name the real affected objects before abstract campaign type.
2. Decision usefulness: every card must answer “so what?” in one or two lines.
3. Evidence separation: fact, interpretation, and missing confirmations stay separate.
4. Thread seed is pre-brief, not menu: it should contain a deeper grounded brief plus starter verbs.
5. No new facts: all content must derive from campaign/undercurrent/opportunity/claim/gap/source health artifacts.
6. Review-only: no execution semantics.

## Decision Drivers

1. The user consumes this primarily in Discord, not terminal.
2. Current board text creates attention load instead of reducing it.
3. Existing Phase 1-8 artifacts contain enough symbols/claims/gaps to produce better summaries.

## Selected Plan

Implement finance-local surface quality improvements:
- humanize raw invalidator/campaign titles
- derive affected object summaries from linked opportunities, atoms, claims, gaps, and source refs
- add `operator_brief` to CampaignProjection
- add `directional_implication` and `affected_objects`
- change board rendering to use object-first card lines
- enrich campaign cache with prepared deep-dive cards for `why`, `challenge`, `sources`, `trace`, `expand`
- improve thread seed content generated by board delivery wrapper for campaign threads

## Rejected Options

Rejected: LLM rewrite of board cards | would add nondeterminism and risk unsupported claims.

Rejected: Wait for parent runtime adapter | content quality is finance-local and should be fixed before more runtime work.

Rejected: Keep raw internal handles but add docs | user experience remains bad.

## Required Output Quality Rules

Campaign board card must include:
- human title with concrete object(s)
- `Implication`: who/what this is good/bad for, if derivable
- `Why now`: evidence/count/source movement in human language
- `Verify first`: one or two concrete checks
- `Known unknown`: top context gap, if present

Thread seed must include:
- short conclusion
- fact bullets
- interpretation bullets
- top unknowns/context gaps
- starter queries

Forbidden in primary board/thread seed:
- raw `direction_conflict:theme:*`
- raw `invalidator direction conflict:*`
- generic “attention slot” without affected object/context
- raw hashes/packet ids
- execution language
